1. Pourquoi organiser son code en modules ?

Le but est de limiter d'exposer ses variables et ses fonctions pour :

  1. Éviter les conflits (et les bugs) ;
  2. Améliorer la maintenabilité ;
  3. Permettre la réutilisation de projet en projets ;
  4. Expliciter les options.

1A. Éviter les conflits... entre fonctions et variables

// A function function foo1() {} // A string, with a conflicted name var foo1 = 'myString'; console.log(typeof foo1);

1B. Éviter les conflits... entre fonctions et variables

for (var i=0; i<10; i++) { console.log(getRandomKey(16)); } function getRandomKey(length) { key = ''; for (i=0; i<length; i++) { key += Math.floor(Math.random() * 10); } return key; }
